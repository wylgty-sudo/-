## 一、产品概述

一款运行在浏览器中的个人助理 Web 应用。用户通过手机或电脑输入待办事项、灵感和素材，系统将内容分类管理，支持四象限优先级排序，提供日/周复盘视图。多设备实时同步，免安装，添加到主屏幕后体验接近原生 App。

**目标用户**：个人用户（单人使用）
**访问方式**：固定 URL（如 `https://your-app.vercel.app`），手机可添加到主屏幕

---

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 + 后端 | Next.js 14 (App Router) | 部署到 Vercel 免费套餐 |
| 数据库 | Supabase PostgreSQL | 免费套餐，500MB |
| 文件存储 | Supabase Storage | 免费套餐，1GB，用于存储截图 |
| 实时同步 | Supabase Realtime | 多设备自动同步 |
| 身份认证 | Supabase Auth + Google OAuth | 一键 Google 登录 |
| 样式 | Tailwind CSS | 响应式，移动优先 |
| 链接预览 | Open Graph 解析（服务端 API Route） | 抓取链接标题和缩略图 |
| 语音输入 | Web Speech API（浏览器原生） | 无额外费用 |

**数据流**：浏览器 ↔ Vercel (Next.js) ↔ Supabase

## 三、功能模块

### 3.1 五大内容模块

| 模块 | 用途 | 特殊功能 |
|------|------|----------|
| 今日待办 | 当天需完成的任务 | 四象限排序、截止日期、自动顺延 |
| 待处理清单 | 有空要做、不紧急的事 | 手动排序 |
| 选题灵感 | 内容创作想法 | 卡片瀑布流 |
| 素材收集 | 链接、截图、参考资料 | 链接预览、图片上传 |
| 日常习惯 | 固定重复事项 | 打卡日历、连击天数 |

### 3.2 今日待办 · 四象限

```
┌─────────────────┬─────────────────┐
│  重要 & 紧急     │  重要 & 不紧急   │
│  （立即做）      │  （计划做）      │
├─────────────────┼─────────────────┤
│  不重要 & 紧急   │  不重要 & 不紧急 │
│  （委托/快速做） │  （考虑删除）    │
└─────────────────┴─────────────────┘
```

- 新建任务时选择所在象限
- 可拖拽跨象限移动
- 支持设置截止日期：临近当天橙色高亮，过期变红
- **顺延规则**：未完成任务次日自动出现在同一象限，直到用户"完成"或"取消"
- "取消"与"完成"是两种不同状态，均进入历史记录

### 3.3 素材收集 · 富媒体支持

- **链接**：粘贴 URL 后服务端自动抓取 Open Graph 数据（标题、描述、缩略图），支持小红书、微博、B站、微信公众号等主流平台
- **截图/图片**：上传后存储到 Supabase Storage，卡片展示缩略图
- **文字**：普通备注文字

### 3.4 日常习惯 · 打卡

- 每个习惯显示本月打卡日历（类似 GitHub contributions 热力图）
- 连续打卡天数展示，形成正向激励
- 当天未打卡的习惯在首页顶部提醒

### 3.5 快速添加

- 任意页面右上角固定 `+` 按钮
- 点击弹出半屏输入框（移动端）/ 居中弹窗（桌面端）
- 底部 5 个分类按钮一触即选
- 麦克风按钮：点击录音 → 自动转文字填入输入框 → 用户可编辑校对 → 确认保存
- 链接识别：检测到 URL 自动切换为素材收集模式并触发预览抓取

### 3.6 搜索

- 顶部常驻搜索栏
- 全局搜索所有模块的标题和内容
- 结果显示来源模块标签

## 四、复盘视图

### 日视图

- 展示指定日期的所有记录（含已完成、已取消）
- 已完成：删除线 + 灰色
- 已取消：删除线 + 更浅灰色 + "取消"标签
- 未完成：正常显示
- 顶部显示当日完成统计（完成 X / 共 Y 项）

### 周视图

- 7 天横向时间轴
- 每天显示完成数 / 总数
- 点击某天展开查看详情

### 统计

- 今日完成数、本周完成数
- 各模块任务数量分布饼图
- 习惯连击天数排行

## 七、数据模型（简要）

```sql
-- 用户（由 Supabase Auth 管理）
users: id, email, created_at

-- 记录（所有模块共用一张表，通过 category 区分）
items:
  id, user_id, category (today_todo | backlog | inspiration | material | habit)
  title, content, status (active | completed | cancelled)
  quadrant (1-4, 仅 today_todo 使用)
  due_date (仅 today_todo 使用)
  original_date (首次创建日期，用于顺延追踪)
  media_url (截图存储路径)
  link_url, link_title, link_image (链接预览数据)
  created_at, updated_at, completed_at

-- 习惯打卡记录
habit_logs: id, item_id, user_id, checked_date, created_at

-- 访问白名单（邀请制）
allowed_users: id, email, invited_by (admin user_id), created_at
```

## 八、范围边界

**本期包含**：以上所有功能，以及：
- **邀请制访问控制**：管理员在设置页面维护 Gmail 邮箱白名单，白名单外的账号登录后显示"暂无访问权限"页面；管理员可随时添加或移除成员；每位成员拥有完全独立的数据空间

**本期不包含**：
- 多用户协作（各用户数据互相隔离，不共享）
- 提醒推送通知（浏览器通知 API 可作为后续迭代）
- 数据导出（可作为后续迭代）
- 深色模式（可作为后续迭代）
